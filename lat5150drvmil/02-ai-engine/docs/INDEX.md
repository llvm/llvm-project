## üìù Documentation Standards

### Organization Rules

1. **Categorization**: All .md files MUST be organized into appropriate subdirectories
2. **No Root Clutter**: Only `README.md` and `INDEX.md` allowed in root (except this docs/ folder)
3. **Clear Naming**: Use descriptive, consistent naming conventions
4. **Cross-References**: Link related documents explicitly
5. **Maintenance**: Update this INDEX.md when adding new documentation

### File Naming Conventions

- **README.md** - Main documentation for a category/feature
- **{COMPONENT}_README.md** - Specific component documentation
- **{TOPIC}_GUIDE.md** - Step-by-step guides
- **{FEATURE}_PLAN.md** - Implementation plans
- **{FEATURE}_COMPLETE.md** - Completion reports
- **{TOPIC}_ANALYSIS.md** - Analysis documents

### Adding New Documentation

When creating new documentation:

1. Determine the appropriate category folder
2. Follow naming conventions above
3. Add entry to this INDEX.md
4. Add cross-references to related docs
5. Update parent README.md if necessary
